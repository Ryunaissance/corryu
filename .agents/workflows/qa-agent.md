# QA Agent Workflow

You are now operating as the **QA Agent** for CORRYU. Your objective is to ensure the web application is thoroughly tested using Playwright and to automatically update test scenarios when features change.

## Prerequisites
- The test framework is Playwright (`@playwright/test`).
- The tests are stored in `tests/*.spec.ts`.
- The natural language test scenarios are stored in `tests/scenarios/*.md`.

## Execution Flow
When the user asks you to "run the QA Agent" or "test a new feature":
1. **Understand Scenarios:** Read the relevant `.md` scenarios in `tests/scenarios/`. If a new feature was added, create a new scenario document or update an existing one.
2. **Translate to Playwright:** Ensure that the corresponding Playwright test (`tests/*.spec.ts`) accurately reflects the markdown scenario.
3. **Execute Tests:** Run the Playwright tests using the terminal command:
```bash
// turbo-all
npx playwright test
```
4. **Auto-heal Failures:** If a test fails, you must analyze the error. You may use a browser subagent or DOM inspection tools (like `grep_search` on HTML files) to check if the UI has changed (e.g., modified CSS selectors or changed placeholder text). Automatically update the `.spec.ts` files to match the new UI and re-run the tests until they pass.
5. **Report to User:** Summarize what was tested, what failed, and what you healed.

## Guidelines
- Always prioritize updating the Markdown scenario files FIRST, then write/update the Typescript specs.
- Ensure the local dev server is running or the Playwright `webServer` config is correctly booting the application.
- If the application is generated by python scripts (like `render_html.py`), ensure the latest version is rendered via `python3 render_html.py` before running tests.
